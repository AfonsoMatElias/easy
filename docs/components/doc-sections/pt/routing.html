Roteamento (Routing)

O roteamento é uma funcionalidade que habilita o modo SPA (Single Page Application), e para criar um SPA com o Easy é muito simples, porque ele uso os elementos do includer.

Configurando (Setting)

No roteamento, o elemento onde os components começam a ser renderizados deve ser especificado, para isso, a directiva/atributo route-view é usado, e na definição dos components algumas propriedades específicas para o roteamento devem definidas, que são:

  • config.base – usado para definidar a URI base do projecto. exemplo: www.dominio.com/meu-projecto/index.html. O valor padrão é /, significando que é a pasta raiz do projecto.
  • config.usehash – usado para permitir a navegação baseado em hash (#). O valor padrão é true.
  • Até este ponto vamos assumir que você já tem conhecimento do que o quê é 'nameId'.
  • nameId.title – representa o título que será configurado quando o component for a adicionado renderizado (É opcional).
  • nameId.route – representa a rota de navegação (É obrigatório no tópico de roteamento).
  • nameId.isDefault – configura o componente como a rota padrão da aplicação.
  • nameId.isNotFound – configura o componente como a rota `not found (404)` da aplicação.
  • nameId.children – representa os filhos deste componente.
  • nameId.keepAlive – perminet manter vivo o estado antigo do componente.

Sempre que se estiver a usar o roteamento, é obrigatório definir a rota padrão para que seja carregado imediatamente. Para isso, configura a propriedade isDefault:true no componente registrado.

No caso da página 404, podemos configurar a propriedade isNotFound:true, dizendo ao Easy que, se por acaso não encontrar nenhum componente com esta rota, insere este componente.

Dois ou mais componentes não podem ser definidos com a propriedade isDefault ou isNotFound, se for, um erro será lançado dizendo `It is not allowed to define more than one 'default'/'notFound' page...`.

Tradução: Não é permitido definir mais de uma página 'default'/'notFound'.

Navegação (Navegation)

A navegação entre páginas é feito com a tag a mas com o atributo :href ou i:href, também pode ser feito com href mas não é recomendado usá-lo, iremos descrever o comportamento de cada um deles.

Ambos :href e i:href qunado definidos criam o atributo `href` combinando com config.base, e sempre que é clicado, a ação padrão da âncora é previnido.

A diferença entre eles é que, usando o atributo :href, a classe active-link é adicionada no elemento que foi clicado, e usando o i:href, a adição da classe é ignorada.

Aninhando Rotas (Nesting Routes)

Filhos podem ser adicionados para cada componente, isto em uma propriedade `children`, e também podemos aninhar rotas... A propridade espera um objecto de component (components.elements) tendo o `nameId` do filho e o objecto.

Note que na rota do filho, pusemos apenas /nomeRotaFilho, porque depois de ser registrado vai se tornar /nomeRotaPai/nomeRotaFilho.
O aninhamento não tem limite, significando que também podemos aninhar dentro do filho.

Parâmetros nas Rotas (Route Params)

Para definir parâmetro na rota, usamos `:nomeParametro` e para extrair dados do parametro da rota, usamos this.getParams() na instânica do componente, retornando um objecto com todos os parametros da rota.

Modo HTML5 / URL Bonitas

O modo padrão de navegação é o modo hash (#), que usa o hash na URL para navegar entre as páginas em recarregar a mesma.

Para sem usar a hash na URL, há alguns passos que devem ser seguido, que são:

  1. Desative o config.usehash – configurando para false isto fará com que o hash não seja adicionado nas âncoras de navegação.
  2. Adicione a tag base no head da sua página – a base precisa ser adicionada com o mesmo valor que config.base da configuração de componentes. Que vai especificar a URL base para usar em todos as URL relativa no documento (DOM). Clique em mim para saber mais sobre a tag base.
  3. Configure o retorno no servidor para ser index.html se a URL não coincide com algum ficheiro estático – já que a aplicação corre em uma só página o `index`, precisamos dizer ao servidor retornar este ficheiro, permitindo que o easy.js lide com a navegação no front-end.
