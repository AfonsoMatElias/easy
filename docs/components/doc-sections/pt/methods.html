Métodos (Methods)

Há alguns métodos internos que podem ser usados na aplicação, que são:

setData

É usado para definir propriedades num escopo. Sua sintâxe é instance.setData(object, targetObject).

  • { Object } object – representa o objecto contendo as propriedades que devem ser adicionadas no escopo.
  • { Object } targetObject – representa o escopo alvo onde as propriedades serão adicionadas. É opcional, se não for definida será posto no escopo principal.
  • { Object } returns – o escopo.

on

Permite adicionar eventos na instância para que possam ser usadas em qualquer lugar da aplicação. O método espera dois parâmetros: instance.on(eventName, callback).

  • { String } eventName – nome do evento a ser adicionado.
  • { Function } callback – o callback tendo seu código.
  • { Object } returns – um objecto tendo o callback que poder ser usado para remover o evento. { event: String, callback: Function } .

off

Permite remover um evento adicionado a instância. O método espera dois parâmetros: instance.off(eventName, callback).

  • { String } eventName – nome do evento a ser removido.
  • { Function } callback – o callback retornado do método on.
  • { Object } returns – um objecto tendo nome do evento e o callback. { event: String, callback: Function }.

emit

Permite você acionar um evento especifico, seja ele customizado ou adicionado a instância. O método espera três parâmetros: instance.emit(eventName, data, once).

  • { String } eventName – nome do evento a ser emitido.
  • { Object } data – dados que seram adicionados no callback.
  • { Boolean } once – permite remover o evento depois de ser emitido a primeira vez.

expose

Permite expôr um objecto, permitindo obter o mesmo em qualquer parte da aplicação. Por padrão, o Easy usá-o para passar dados para um componente. Espera três parâmetros: instance.expose(name, data, force).

  • { String } name – um nome único(Ou, nome do componente).
  • { Object } data – o objecto a ser exposto.
  • { Boolean } force – permite forçar a exposição se por acaso existe um objecto já exposto com o mesmo nome.

retrieve

Permite-nos obter os dados expostos. O método espera dois parâmetros: instance.remove(name, remove).

  • { String } name – nome do objecto exposto.
  • { Boolean } remove – permite-nos remover os dados expostos depois de obtê-los.
  • { Object } return – o objecto exposto.

toJsObj

Permite-nos criar um objecto javascript estruturado, baseando-se na estrutura do elemento HTML. Objecto esse que pode ser enviado para um servidor.

Para ser capaz de gerar a estrutura este método depende de alguns comandos, que são:

  • e-build – permite construir um objecto dentro do outro. A sintâxe é e-build="caminho". O caminho precisa ser sempre completo, do base até a propriedade que precisa ser criada. Quando precisar criar um objecto dentro de um array não precisa se preocupar com índice, defina apenas o nome, por padrão ele irá mapear todo array.
  • e-array – permite criar um array do elemento onde foi definido. Mesmo que o atributo não foi definido, quando o elemento se repete ele cria um array dos elementos.

O método espera três parâmetros: instance.toJsObj(selector, options, onset).

  • { String | Element } selector – o selector do elemento a ser gerado ou o elemento em si.
  • { Object } options – opções que indicam em que atributos precisam ser procurados e deste que attributos o valor precisa ser pegue. A sintâxe é { names: '[atributo1],[atributo2]', values: '[atributo1],[atributo2]' }. A ordem de definição é muito importante, ele pega o que encontrar primeiro. O valor padrão é { names: '[name]', values: '[value]' }.
  • { Function} onset – a função que será chamada quando uma propriedade é configurada.

Podemos construir objectos complexos usando este método, só precisamos criar a nossa estrutura.

watch

É usado para assistir (escutar) uma propriedade específica do escopo. A sintâxe é: instance.watch(propertyName, callback, targetObject).

  • { String } propertyName – o nome da propriedade do escopo.
  • { Object } callback – representa o callback onde terá toda lógica que será executada quando a propriedade mudar. O primeiro argumento é a novo valor e o segundo argumento é o antigo valor da propridade.
  • { Object } targetObject – representa o escopo que tem a propriedade que será assistida (escutada). É opcional, se não for definido vai assumir o escopo principal.
  • { Object } returns – a instância do watch, que pode ser usado para destruí-lo.

set

Usado para mudar algumas configurações na instância Easy. O método espera dois parâmetros: O primeiro é a propriedade a ser modificada, e o segundo o valor (Object Literal) a ser configurado. As configurações disponíveis são:

  • skeleton – permite modificar o valor de `background` e ou de `wave` do skeleton. A sintâxe é instance.set('skeleton', object).
    • { Object } object – o objecto tendo o(s) valor(es) a ser modificado. { background: 'cor', wave: 'cor' }
  • delimiter – permite adicionar um delimitador no compilador do Easy. A sintâxe é instance.set('delimiter', object).
    • { Object } object – o objecto tendo os delimitador a ser adicionado. A estrutura do objecto é:

    Todos os delimitadores disponíveis podem ser vistos na propriedade instance.delimiters. Estes valores são inalteráveis usando esta propridade.

    Lembre de escapar valores de `open` e `close` se preciso, porque eles passam por um Expressão Regular.
