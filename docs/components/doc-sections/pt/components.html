Componentes (Components / Includers)

É um conceito que permite a reutilização do código HTML. Easy cria e manipula componentes de uma maneira diferente, veremos nos capítulos abaixo...

Os termos Includer e Component no Easy.js é a mesma coisa, portanto, quando dissermos `includer` significa que estamos a nos referir a um `component` e quando dissermos `component` também estamos a nos referir a um `includer`.

Há dois tipo de componentes, Clonado (Cloned) e Registrado (Registered), e os dois podem ser importado/incluidos do mesmo jeito, usando a tag inc ou usando um Elemento Oficial do DOM (um elemento container).

Clonado (Cloned)

Esses são componentes que se baseiam na cópia de um elemento durante a compilação, é como copiar e colar. Pode ser alcançado definindo o atributo inc-tmp="nameId" (significa include template) no elemento que precisa ser copiado e incluir onde o mesmo deve ser colado, mas define @ antes do valor do `nameId`, isto diz ao Easy é um elemento clonado.

Registrado (Registered)

Estes são os componentes que são registrados directamente da instância, isto pode ser feito na Classe do Easy ou usando o objecto de gestão de componentes na instância instance.components.

Registering (Registrando)

Há duas formas de registrar um componente, que são: Forma Curta, onde definimos o nameId com o caminhoDoComponente e a Forma Completa, onde definimos o nameId com o objectoDeOpcoes.

Em Easy Options, o objeto `components` tem a seguinte estrutura:

  • config.keepData – usado para permitir manter todas as alterações feitas nos dados do componente. O valor padrão é false.
  • elements – usado para definir todos os elementos de componentes. On na esquerda vai o nameId, e na direita vai String caminhoDoComponente (do servidor) ou o Objecto de Opções.
  • nameId.url – representa do caminho do component. Isto usa o config.base para requisitar o ficheiro do servidor. A extensão .html é opcional.
  • nameId.template – representa o código html bruto para o modelo. Não use junto com nameId.url.
  • nameId.restrictions – permite addicionar verificações antes do component ser requisitado. Isto espera uma array de functions que retornam um Boolean, true para permitido, e false para não permitido.
  • nameId.store – permite armazenar o component quando o mesmo for requisitando a primeira vez. Por padrão o valor é true.
  • nameId.data – representa os dados que será injectado no processo de compilação do componente.

Lidando com Components (Handling Components)

O objeto instance.components permite-nos gerir components. Ele está estruturado da segunte forma:

  • add – é um método que permite adicionar components na instância. Ele espera um objecto tendo tenho o elemento do component: add({...}).
  • get – é um método que permite obter o objecto de opções do component. Espera o nameId do componente: get(nameId).
  • restrictions – é um objecto que permite gerir as restrições do componente.
  • restrictions.add – permite adiciona uma restrição a um componente. Espera dois parâmetros: restrictions.add(nomeComponente, arrayRestricoes).
  • restrictions.get – permite pegar as restrições de um component. Espera o nameId do componente: restrictions.get(nameId).
  • restrictions.remove – permite remover uma restrição de um componente. Espera dois parâmetros: restrictions.add(nomeComponente, funcaoRestricao).

Formato do Componente (Component Format)

Eles são gravados em arquivos .html, contendo estilos (opcional), scripts (opcional) e um elemento raiz (obrigatório).

Estilos (Styles)

A tag de estilo (style) aqui funciona como esperado, por padrão os estilos são globais para definir que tem um escopo, basta adicionar o atributo scoped. The style tag here works as expected, by default the styles is global to define it has a scope one just add the scoped attribute. Isso adiciona uma classe como easy-sAb_c123 às regras do estilo.

Elemento Raiz (Root Element)

Um elemento de bloco como um div que será adicionado ao DOM. Se não foi definido um erro será mostrado dizer: The component 'nameId' seems to be empty or it has not a root element, e.g.: <div></div>, to included.

Tradução: O componente 'nameId' parece estar vazio ou não tem um elemento raiz, exemplo: <div></div>, para ser incluido.

Scripts

Um bloco que contém o código JavaScript que é executado quando o componente é criado. O contexto (this) é a instância do componente.

O palavra reservada this permite-nos ter acesso aos dados do component acessando this.data. Para ter acesso aos dados do componente acima (outside scope), usamos this.scope.

A instância do Easy deste componente pode ser acessado em this.Easy.

Hooks do Ciclo de Vida (Lifecycles Hooks)

São funções que são chamadas em alguns estagios de vida do componente, em que na qual podemos citar:

Criado (Created)

Este é o hook que é chamado quando o componente foi criado. Pode ser definido da seguinte forma: this.created = function(element){...}. O elemento raiz é passado como o primeiro parâmetro.

Montado (Mounted)

Este é o hook que é chamado quando o componente foi montado ou adicionado no DOM. Pode ser definido da seguinte forma: this.mounted = function(element){...}. O elemento raiz é passado como o primeiro parâmetro.

Carregado (Loaded)

Este é o hook que é chamado quando o componente foi carregado ou compilado. Pode ser definido da seguinte forma: this.loaded = function(element){...}. O elemento raiz é passado como o primeiro parâmetro.

Destruido (Destroyed)

Este é o hook que é chamado quando o componente foi destruido ou removido do DOM. Pode ser definido da seguinte forma: this.destroyed = function(element){...}. O elemento raiz é passado como o primeiro parâmetro.

É recomendado pôr toda requisição de dados (fetch) dentro dos hooks created or mounted, de acordo a velocidade da resposta, o component poderá ser mostrado já com os dados.

Exportando Dados (Exporting)

A função que é usada para exportar dados do script para o modelo (template) é export. Para ser capaz de exportar os dados, ele precisa ser definido como a seguir: this.export({...}).

Todos os dados se tornam reactivos após serem exportados.

Os hooks não são obrigatórios, pois podem aparecer no exemplo. No final, um componente tem o seguinte formato:

Inc com data (Inc with data)

Um includer poder ser combinado com o comando data para passar dados dentro do componente. Este This data can be accessed in the property this.data dentro do script do componente.

Os attributos definidos no elemento do includer (inc) são transfiridos para elemento do template ou elemento raiz (exemplo.: div) do component.

Inc com Delimitador (Inc with Delimiter)

Um includer pode ser combinado com um delimitador, oferecendo mudanças dinâmicas quando os valores da propriedade mudam. Só um delimitador pode ser passado.

Inc com `on` Método (Inc with `on` Method)

Existem eventos integrados para includers que nos permitem escutar estados diferentes de um includer, que são:

  • incRequested – este evento é despachado quando o DOM solicita um includer.
  • incMounted – este evento é despachado quando um includer é montado no DOM.
  • incLoaded – este evento é despachado quando um includer é carregado (compilado) ou compilado.
  • incDestroyed – este evento é despachado quando um includer é destruído.
  • incBlocked – este evento é despachado quando um includer é bloqueado por razões de restrições.
  • incFail – este evento é despachado quando um includer falha na requisição.

Keep Alive

Por padrão, os componentes são recriados quando são solicitados a serem incluídos, para manter o elemento com as informações basta definir o keep-alive. Os dados no script será recarregado.

Aninhando Conteúdo (Nesting Content)

O conteúdo pode ser facilmente aninhado, basta definir uma tag content no corpo do inc, e define uma tag content dentro modelo (template) do component (onde o conteúdo será inserido).

Que tal múltiplos conteúdos (What about multiple contents)

Neste caso adicionamos o atributo ident na tag content, isto faz com que o Easy identifique onde cada conteúdo precisa estar.

Aninhando Includers (Nesting Includers)

Os Includers podem ser facilmente aninhados, seguindo a mesma linha de pensamento comNesting Content, just put inc tag inside the content tag.

Todos os includer são compilados automaticamente, o que significa que após serem adicionados ao DOM serão compilados.
