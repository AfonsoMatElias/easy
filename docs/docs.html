Introduction

What is Easy.js?

Easy.js, short for Easy and Asyncronous Javascript, is a javascript library for building user interfaces and that helps in the development of web applications, providing a synchronous interaction between user interfaces and Javascript data.

So, Why Easy.js?

As the name says, it is a javascript library really easy to use that provides to you a simple way to interact with your HTML by using Reactive Properties, Scoping data in the HTML, reusable components written in pure HTML, api calls using only attributes, and so on... It's very good and simple to build Single Page Application. No Virtual DOM is used, it uses the Real DOM to handle all the data changes.

Browsers Compatibility

It was tested in almost every popular browsers and IE 11, below that is not suported, because it uses some comcepts of ECMAScript 2015 (ES6) .

Scale of Projects

A web application fully developed with easy.js was tested only with small projects, larger projects was not tested yet.

Installation

It is easily added to a web project, just like the other libraries and framework, and for that there are two ways.

Locally

To use it locally, you need to download the file and import it

Download 


    <script src="/js/easy.js"></script>

Or, you can use...

CDN

Using CDN is a little bit simpler because the only thing that needs to be done is to import it 


    <script src="https://cdn.jsdeliv.com/gh/afonsomatelias/2.0.0/easy.js"></script>

Instance

When you import Easy it will added a function to the global object, so, to begin using it you need to create a new instance of Easy Class, this instance expects two parameters: 


    var app = new Easy(element, options);

Element

The element is the mandatory parameter that represents the section that the instance will control in the application, it must be passed as a String in the format of a CSS Selector. #myId, .myClass or body.

Options

Options is the optional parameter, which can be used to define initial data, instance configuration, components (we'll talk about this later), etc. If passed, it must be an object containing all the information you need when the instance is created.

This property can be accessed from the instance object: instance.options.

Properties

As previously stated, in the options parameter several properties can be defined which are: 


    var app = new Easy('body', {
        // The configuration of the instance
        config: { ... },

        // The initial data of the instance
        data: { ... },

        // The function that will be call when the DOM is loaded
        mounted: function (el) { ... },

        // The function that will be call when the Easy compile the code
        loaded: function (el) { ... },

        // The components that the instance will inject in your app (we'll talk about this later)
        components: { ... }
    });

Config Property

With config property you may set:

  • { Boolean } log – Allows the logs (logs, errors and warning) to be shown. Default value is true.
  • { Boolean } deepIgnore – Allows you to ignore all elements that are inside a container (even those that are added later) when using the command e-ignore (We'll talk about that in the Commands section).
    Be careful, this can slow down your application.
  • { Boolean } useDOMLoadEvent – Allows the instance to use the 'DOMContentLoaded' event when loading the page. Default value is true.
  • { Object } skeleton – Defines color of the skeleton while the content is being loaded. Is structured as follows:
    { background: 'color', wave: 'color' }
    By default the skeleton is light gray with the lightest wave.

Data

As already mentioned, the data property allows you to define the data that the application will have when it starts, however there is a way to add data while it is already running with the app.setData(...) method. (We'll talk about that in the methods section).
Data or properties defined on date are not normal properties, they are reactive, this means that they react to changes made to them. 


    <div id="app1">
        <h4> {{ message }} </h4>
    </div>
                            

    var app1 = new Easy('#app1', {
        data: {
            message: 'Hello World!'
        }
    });

This property can be accessed from the instance object: instance.data, You can open the browser console and type app1.data.

Instance Hooks

There are only two main hooks in the Easy instance, which are:

  • { Function } mounted – this hook is called when the DOMContentLoaded event is raised (It works this way if the useDOMLoadEvent property is true, otherwise, it will be called as soon as the Class is instantiated).
  • { Function } loaded – this hook is called when the root element was compiled. 


    <div id="app2">
        <h4> Mounted at: {{ mountedHook }} </h4>
        <h4> Loaded at: {{ loadedHook }} </h4>
    </div>
                            

    var app2 = new Easy('#app2', {
        data: {
            mountedHook: '',
            loadedHook: '',
        },
        mounted: function(){
            this.data.mountedHook = getDateTime()
        },
        loaded: function(){
            this.data.loadedHook = getDateTime()
        }
    });

The context of these hooks is the current instance and the root element is passed as an argument.

You can create an instance without a variable, this means that your data cannot be accessed by the browser console.

Delimiters

By default Easy brings two types of delimiters, which are the easy delimiter and the common delimiter that is used by Vue.js, Angular.js, etc.

The delimiters open a space in HTML to execute some javascript code, it means that you can execute any pure js code variables, function, ternary operatiors, etc. I only need to return a value.

Whenever data is an Object it turn it to JSON.

Easy

To use this delimiter just do the following: -e- [Content] -

Common

To use this delimiter just do the following: {{ [Content] }}

You can define your own delimiter, has you wish (we'll talk about this later).

Bindings

One Way

You can achieve One Way Data Binding simply by defining the delimiter in the space or attribute you need the value to be. 


    <div id="app3">
        <input type="text" value="{{ value }}" />
        <h4> {{ value }} </h4>
    </div>
                            

    var app3 = new Easy('#app3', {
        data: {
            value: 'Type something!'
        },
        loaded: function(el) {
            var self = this;
            el.node('input').onkeyup = function(evt) {
                self.data.value = evt.target.value;
            }
        }
    });

If you don't want to do it this way, you can still use the command e-property="{{ dataProperty }}", it creates the property after being compiled. This commands is very versatile, we will go into details about it in the commands section. 


    <div id="app3">
        <input type="text" e-value="{{ value }}" />
        <h4 e-title="{{ value }}" > Hello! </h4>
    </div>

Two Way

You can achieve Two Way Data Binding using the command e-bind:property="dataProperty", by default it binds value property, it means that if you want to bind the property value of an input you can define like this e-bind="dataProperty". 


    <div id="app4">
        <input type="text" e-bind:value="value" /> <br>
        <input type="text" e-bind="value" />
        <h4> {{ value }} </h4>
    </div>
                            

    var app4 = new Easy('#app4', {
        data: {
            value: 'Type something!'
        }
    });

Commands

Easy has some commands that are used as attributes in elements, each one has its peculiarity and way of using it, we will explain each step by step how and where it can be used.

data

It allows defining a data scope for a certain element, the syntax is data="{ ... }", if it is defined to be content the element will have the data of the main scope, instance.data.

If you define an Object Literal, kwow that these properties are temporary and by default they are not reactive, unless an object that is already reactive is passedby itself or if you passe the object in the this.reactive( ... ) method, it expects a method and returns the already reactive object. 


    <div id="app5">
        <div>
            <h4> {{ value }} </h4>
        </div>

        <div data="scope1">
            <h4> 1st {{ value }} </h4>
        </div>

        <div data="scope2">
            <h4> 2nd {{ value }} </h4>

            <div data>
                <h4> I am in Scope 2 element, but, {{ value }}  </h4>
            </div>
        </div>
    </div>
                            

    var app5 = new Easy('#app5', {
        data: {
            value: 'This is a value from main scope üôÇ',
            scope1: {
                value: 'This is a value from 1st scope'
            },
            scope2: {
                value: 'This is a value from 2nd scope'
            }
        }
    });

Now let's see how to create a scope with reactive properties, using the reactive function. 


    <div id="app6">
        <div data="this.reactive(function(){
            return { name: person.name, country: 'Angola' }
        })" >

            <h4> {{ name }} </h4>
            <h4> {{ country }} </h4>

            <br> Lets prove that they are reactives  <br>
            
            <input type="text" e-bind="name" />
            <input type="text" e-bind="country" />

        </div>
    </div>
                            

    var app6 = new Easy('#app6', {
        data: {
            person: {
                name: 'Afonso Matumona'
            }
        }
    });

wait-data

Allows the element where it is defined to be skipped in the compilation, waiting for its data to be provided. It is defined as follows: wait-data="dataID". In order to make the element data available, we use the expose method. 


    <div id="app7">
        <button on:click="load" > Load Remain </button>
        
        <h4>
            {{ message }}! <span wait-data="waitForMe"> {{ message }} </span>
        </h4>
    </div>
                            

    var app7 = new Easy('#app7', {
        data: {
            message: 'Hello',
            load: function(){
                this.expose('waitForMe', { message: 'how are you? üòÄ' });
            }
        }
    });

The data passed it the expose method for wait-data, automatically become reactive, then it is no longer necessary to use the reactive method in this command.

We don't know about you, but for us the result of the example above is kind of ugly with the delimiters in sight, fortunately we have e-content.

e-content

It is a way to define a value that will be put in the container after the element is compiled.

Let's redo the example. 


    <div id="app8">
        <button on:click="load" > Load Remain </button>
        
        <h4>
            {{ message }}! <span wait-data="waitForMe" e-content="{{ message }}"> ... </span>
        </h4>
    </div>
                            

    var app8 = new Easy('#app8', {
        data: {
            message: 'Hello',
            load: function(){
                this.expose('waitForMe', { message: 'how are you? üòÄ' });
            }
        }
    });

This way it looks much better, but we can still improve it, let's combine it with a skeleton.

e-skeleton

It is a way of showing that information from a specific area is being loaded, it is used a lot in today's applications.

Please don't judge us by this example, it may not even make much sense, but it's just to show you how it works üòÄ.

Now, let's redo the example above. 


    <div id="app9">
        <button on:click="load" > Load Remain </button>
        
        <h4>
            {{ message }}! 
            <span wait-data="waitForMe" e-skeleton e-content="{{ message }}" > Waiting </span>
        </h4>
    </div>
                            

    var app9 = new Easy('#app9', {
        data: {
            message: 'Hello',
            load: function(){
                this.expose('waitForMe', { message: 'how are you? üòÄ' });
            }
        }
    });

When e-skeleton is defined for an element, it takes the form of the element to be able to present itself, this means that the element must have some width and height. And by default the background color is gray and the wave is light gray.

There are two ways to use it, e-skeleton and e-skeleton="someId", Without an identifier, the skeleton is removed after the element has been compiled. With an identifier, the skeleton(s) must be removed manually using element.clearSkeleton(id) extension method or simply remove the e-skeleton attribute from the elements. 


    <div id="app10">
        <h4 e-skeleton="skeleton1"> {{ message }}! </h4>
        
        <button on:click="removeSkeleton" > Remove Skeleton</button>
    </div>
                            

    var app10 = new Easy('#app10', {
        data: {
            message: 'Hello',
            removeSkeleton: function(){
                this.el.clearSkeleton('skeleton1');
            }
        }
    });

e-def

It is a short name of e-definer, it allows to dynamically define properties (Reactives) in the current scope. Its syntax is e-def="{...}", it expects Object Literal. 


    <div id="app11">        
        <div e-def="{ myProp: 'I was created dynamically' }"> 
            {{ myProp }}! 
        </div>
    </div>
                            

    var app11 = new Easy('#app11', {
        data: {}
    });

The order of definition matters a lot, e-def must always be above the elements where it will be used

e-if

The e-if command is very similar to the Vue.js v-if directive... it is used to conditionally render a block (element). The block will only be shown if the command expression returns a truthy value. It is accompanied with the e-else-if and e-else commands. 


    <div id="app12">
        <input type="number" e-bind="binary" /> <br> 
        
        <p e-if="binary == 1"> 
            {{ binary }} is on üôÇ
        </p>
        <p e-else-if="binary == 0"> 
            {{ binary }} is off üôÅ
        </p>
        <p e-else> 
            {{ binary }} is not binary ü§®
        </p>
    </div>
                            

    var app12 = new Easy('#app12', {
        data: {
            binary: 1
        }
    });

e-show

The e-show command is very similar to the Vue.js v-show directive... it is used to conditionally show an block (element), it toggles CSS display property. The block will be visible if the command expression returns a truthy value. 


    <div id="app13">
        <button on:click="show = !show"> toggle {{ show ? 'üôâ' : 'üôà' }} </button>

        <div e-show="show"> 
            <p> Want to hide me? Click on toggle</p>
        </div>
    </div>
                            

    var app13 = new Easy('#app13', {
        data: {
            show: true
        }
    });

e-for

The e-for command is very similar to the Vue.js v-for directive... It is used to render a list of items based an array. Can be used in two ways, with declared variable(s) or not.

There are two syntaxes for its use, the simple e-for="myArray" normally used when it is an array of object literal, and the complete e-for="item of/in myArray" used mandatorily when it is an array of primitives and optional when they are object literal. 


    <div id="app14">
        <h4> Primitives</h4>
        <ul> 
            <li e-for="fruit of fruits" > {{ fruit }} </li>
        </ul>
        
        <h4> Objects</h4>
        <ul> 
            <li e-for="foods" > {{ name }} </li>
        </ul>
    </div>
                            

    var app14 = new Easy('#app14', {
        data: {
            fruits: [ 'Bananas', 'Mango' ],
            foods: [ { name: 'Rice' }, { name: 'Beans' } ],
        }
    });

Using Array filter

Easy array filter is a way to filter data while it is being rendered. To add the filter we use pipe character ( | ) after e-for expression. item of items | filterExpression. There is four ways to use this filter, which are:

1. Global Fitler

Performs the filter on every property of the items containing the value of the data property set. 


    <div id="app15">
        <input type="text" 
                e-def="{ search: '' }" 
                e-bind="search" 
                placeholder="Search for food" />
        
        <ul> 
            <li e-for="food of foods | search"> 
                {{ food.name }} 
            </li>
        </ul>
    </div>
                            

    var app15 = new Easy('#app15', {
        data: {
            foods: [ { name: 'Rice' }, { name: 'Beans' } ],
        }
    });

2. Targeted filter

Performs the filter on specific properties of the items containing the value of the data property set. The filter need to be defined as following: | dataProperty:property1,property2... 


    <div id="app16">
        <input type="text" 
                e-def="{ search: '' }" 
                e-bind="search" 
                placeholder="Search for food" />
        
        <ul> 
            <li e-for="food of foods | search:name,price"> 
                {{ food.name }}, {{ food.price }} üí∞ 
            </li>
        </ul>
    </div>
                            

    var app16 = new Easy('#app16', {
        data: {
            foods: [ { name: 'Rice', price: 10 }, { name: 'Beans', price: 15 } ]
        }
    });

3. Direct filter

Performs the filter on specific property of the items to a value set. The filter need to be defined as following:
| :property1 === someValue 


    <div id="app17">
        <ul> 
            <li e-for="food of foods | :price >= 15"> 
                {{ food.name }}, {{ food.price }} üí∞ 
            </li>
        </ul>
    </div>
                            

    var app17 = new Easy('#app17', {
        data: {
            foods: [ { name: 'Rice', price: 10 }, { name: 'Beans', price: 15 }, { name: 'Meat', price: 20 } ]
        }
    });

4. Function filter

Performs the filter of the items based on a function, the context (this) of the funciton is the instance, the array is the first argument of the funciton. It expects an array as return value. The filter need to be defined as following:
| :myFunction 


    <div id="app18">
        <ul> 
            <li e-for="food of foods | foodFilter"> 
                {{ food.name }}, {{ food.price }} üí∞ 
            </li>
        </ul>
    </div>
                            

    var app18 = new Easy('#app18', {
        data: {
            foods: [ { name: 'Rice', price: 10 }, { name: 'Beans', price: 15 }, { name: 'Meat', price: 20 } ],
            foodFilter: function(array){
                return array.filter(function(item){
                    return item.price <= 15;
                })
            }
        }
    });

Array Change Detection

Easy links with the main methods of an array, making them become reactive, the methods are:

  • push()
  • pop()
  • unshift()
  • shift()
  • splice()

If you use some method that changes the array even if it is not in the list above the user interface will be updated too.

Easy cannot detect changes when you modify the length of the array. eg.: array.length = newLength . If will not have effects in to the user interface.

e-order

Sorts a list according to the condition passed in the command's expression. Its syntax is e-order="OrderType:PropertyToBeOrdered". The order types are: asc, for ascending and des, for descending. 


    <ul> 
        <li e-for="food of foods" e-order="asc:name"> 
            {{ food.name }}, {{ food.price }} üí∞ 
        </li>
    </ul>

The command accepts delimiters, this means that it can be defined like this, eg: e-order="{{orderType}}:{{orderBy}}"

e-amn

It is a simple way to animate the insertion of an element in the DOM. Its syntax is e-anm="direction".

  • up , Performs the animation starting from the bottom to the top, ascending and gradually increasing the opacity.
  • down , Performs the animation starting from the top to the bottom, descending and gradually increasing the opacity.
  • left , Performs the animation starting from the right to the left, scrolling and gradually increasing the opacity.
  • right , Performs the animation starting from the left to the right, scrolling and gradually increasing the opacity. 

    <ul> 
        <li e-for="food of foods" e-anm="up"> 
            {{ food.name }}, {{ food.price }} üí∞ 
        </li>
    </ul>

e-toggle

Allows you to toggle a property on an element, setting the value to true when set. Its syntax is e-toggle:property="dataProperty", 


    <div id="app19">
        <input type="text" e-toggle:disabled="locked" placeholder="Type your name" />
        <button on:click="locked = !locked"> 
            {{ locked ? 'Locked üîí' : 'Unlocked üîì' }} 
        </button>
    </div>
                            

    var app19 = new Easy('#app19', {
        data: {
            locked: false
        }
    });

e-[something]

As we said above, it is the most versatile command according to the defined expression, because on the one hand it creates a property after being compiled and on the other it creates the property and toggles a value on it.
