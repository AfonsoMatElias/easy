Introduction

What is Easy.js?

Easy.js, short for Easy and Asyncronous Javascript, is a javascript library for building user interfaces and that helps in the development of web applications, providing a synchronous interaction between user interfaces and Javascript data.

So, Why Easy.js?

As the name says, it is a javascript library really easy to use that provides to you a simple way to interact with your HTML by using Reactive Properties, Scoping data in the HTML, reusable components written in pure HTML, api calls using only attributes, and so on... It's very good and simple to build Single Page Application. No Virtual DOM is used, it uses the Real DOM to handle all the data changes.

Browsers Compatibility

It was tested in almost every popular browsers and IE 11, below that is not suported, because it uses some comcepts of ECMAScript 2015 (ES6) .

Scale of Projects

A web application fully developed with easy.js was tested only with small projects, larger projects was not tested yet.

Installation

It is easily added to a web project, just like the other libraries and framework, and for that there are two ways.

Locally

To use it locally, you need to download the file and import it

Download 


    <script src="/js/easy.js"></script>

Or, you can use...

CDN

Using CDN is a little bit simpler because the only thing that needs to be done is to import it 


    <script src="https://cdn.jsdeliv.com/gh/afonsomatelias/2.0.0/easy.js"></script>

Instance

When you import Easy it will added a function to the global object, so, to begin using it you need to create a new instance of Easy Class, this instance expects two parameters: 


    var app = new Easy(element, options);

Element

The element is the mandatory parameter that represents the section that the instance will control in the application, it must be passed as a String in the format of a CSS Selector. #myId, .myClass or body.

Options

Options is the optional parameter, which can be used to define initial data, instance configuration, components (we'll talk about this later), etc. If passed, it must be an object containing all the information you need when the instance is created.

This property can be accessed from the instance object: instance.options.

Properties

As previously stated, in the options parameter several properties can be defined which are: 


    var app = new Easy('body', {
        // The configuration of the instance
        config: { ... },

        // The initial data of the instance
        data: { ... },

        // The function that will be call when the DOM is loaded
        mounted: function (el) { ... },
        
        // The function that will be call when the Easy compile the code
        loaded: function (el) { ... },

        // The components that the instance will inject in your app (we'll talk about this later)
        components: { ... }
    });

Config Property

With config property you may set:

  • { Boolean } log – Allows the logs (logs, errors and warning) to be shown. Default value is true.
  • { Boolean } deepIgnore – Allows you to ignore all elements that are inside a container (even those that are added later) when using the command e-ignore (We'll talk about that in the Commands section).
    Be careful, this can slow down your application.
  • { Boolean } useDOMLoadEvent – Allows the instance to use the 'DOMContentLoaded' event when loading the page. Default value is true.
  • { Object } skeleton – Defines color of the skeleton while the content is being loaded. Is structured as follows:
    { background: 'color', wave: 'color' }
    By default the skeleton is light gray with the lightest wave.

Data

As already mentioned, the data property allows you to define the data that the application will have when it starts, however there is a way to add data while it is already running with the app.setData(...) method. (We'll talk about that in the methods section).
Data or properties defined on date are not normal properties, they are reactive, this means that they react to changes made to them. 


    <div id="app1">
        <h4> {{ message }} </h4>
    </div>
                            

    var app1 = new Easy('#app1', {
        data: {
            message: 'Hello World!'
        }
    });

This property can be accessed from the instance object: instance.data, You can open the browser console and type app1.data.

Instance Hooks

There are only two main hooks in the Easy instance, which are:

  • { Function } mounted – this hook is called when the DOMContentLoaded event is raised (It works this way if the useDOMLoadedEvent property is true, otherwise, it will be called as soon as the Class is instantiated).
  • { Function } loaded – this hook is called when the root element was compiled. 


    <div id="app2">
        <h4> Mounted at: {{ mountedHook }} </h4>
        <h4> Loaded at: {{ loadedHook }} </h4>
    </div>
                            

    var app2 = new Easy('#app2', {
        data: {
            mountedHook: '',
            loadedHook: '',
        },
        mounted: function(){
            this.data.mountedHook = getDateTime()
        },
        loaded: function(){
            this.data.loadedHook = getDateTime()
        }
    });

The context of these hooks is the current instance and the root element is passed as an argument.

You can create an instance without a variable, this means that your data cannot be accessed by the browser console.

Delimiters

By default Easy brings two types of delimiters, which are the easy delimiter and the common delimiter that is used by Vue.js, Angular.js, etc.

The delimiters open a space in HTML to execute some javascript code, it means that you can execute any pure js code variables, function, ternary operatiors, etc. I only need to return a value.

Whenever data is an Object it turn it to JSON.

Easy

To use this delimiter just do the following: -e- [Content] -

Common

To use this delimiter just do the following: {{ [Content] }}

You can define your own delimiter, has you wish (we'll talk about this later).

Bindings

One Way

You can achieve One Way Data Binding simply by defining the delimiter in the space or attribute you need the value to be. 


    <div id="app3">
        <input type="text" value="{{ value }}" /> 
        <h4> {{ value }} </h4>
    </div>
                            

    var app3 = new Easy('#app3', {
        data: {
            value: 'Change me!'
        },
        loaded: function(el) {
            var self = this;
            el.node('input').onkeyup = function(evt) {
                self.data.value = evt.target.value;
            }
        }
    });

If you don't want to do it this way, you can still use the command e-[property]="{{ dataProperty }}", it creates the property after being compiled. This property is very versatile, we will go into details about it in the commands section. 


    <div id="app3">
        <input type="text" e-value="{{ value }}" /> 
        <h4 e-title="{{ value }}" > Hi </h4>
    </div>

Two Way

You can achieve Two Way Data Binding using the command e-bind:property="dataProperty", by default it binds value property, it means that if you want to bind the property value of an input you can define like this e-bind="dataProperty". 


    <div id="app4">
        <input type="text" e-bind:value="value" /> <br> 
        <input type="text" e-bind="value" /> 
        <h4> {{ value }} </h4>
    </div>
                            

    var app4 = new Easy('#app4', {
        data: {
            value: 'Change me!'
        }
    });
